# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

Axhost-Make is a **local-first, Agent-driven prototyping framework**. It provides a Node.js HTTP dev server and a browser-based Shell UI for managing multiple prototype projects — browsing pages/components in a tree view, editing docs, and one-click copying assembled prompts to feed external AI agents.

**Zero external npm dependencies** on the backend. Frontend uses vanilla HTML/CSS/JS + marked.js + Mermaid.js (both MIT-licensed, vendored in `client/assets/`).

## Repository layout

This repo (`axhost-make/`) is the **framework source**. It is typically cloned into a workspace directory where `projects/` (user prototypes) lives alongside it:

```
workspace/
├── axhost-make/   ← this repo (framework)
├── projects/      ← user prototype projects (NOT in this repo)
├── start.cmd / start.sh / package.json  ← workspace-level (generated by init)
```

The workspace-level files (`start.*`, `package.json`) are generated by `bin/axhost-init.js` when running `node axhost-make/bin/axhost-make.js init`. They are NOT part of this repo.

## Key commands

```bash
# Start the dev server (bound to 127.0.0.1 only)
node axhost-make/bin/axhost-make.js serve [--port 3820]

# Start a static preview server (prototype/ only, no API)
node axhost-make/bin/axhost-make.js preview [--port 8080]

# Rebuild templates/preview/ from client/ sources (run after editing preview-index.html, shell.css, icons.js, or preview-app.js)
node axhost-make/bin/axhost-make.js build

# Sync templates + static assets to all workspace projects
node axhost-make/bin/axhost-make.js update --all

# Sync to a specific project
node axhost-make/bin/axhost-make.js update --id <project-hash>

# CLI for project structure (run from project root, not workspace root)
node axhost-make/bin/axhost-make.js add-page <name> [--parent <path>]
node axhost-make/bin/axhost-make.js add-component <name> [--parent <path>]
node axhost-make/bin/axhost-make.js add-folder <name> [-t pages|components] [--parent <path>]
node axhost-make/bin/axhost-make.js add-doc <name> --to <path-or-hash>
```

After editing `server/`: kill the Node process and restart `serve` (no hot reload).
After editing `bin/`: test by running the command directly.
**After editing `templates/` or `client/` (build + update)**: if the dev server was already running, remind the user to restart it — existing processes won't pick up the new files until restarted.

## Architecture

### Server (`server/`)
- `index.js` — `http.createServer` entry point, creates router and starts listening
- `router.js` — URL routing: `/api/*` → API handlers, `/client/*` → static files from `client/`, `/projects/{id}/prototype/*` → project static files, `/prototype/*` → fallback with referer-based project detection
- `api/` — one file per endpoint (scan, file, create, rename, delete, move, copy, reorder, export, docs, settings, projects, open-editor, axhost-proxy, terminal)
- `middleware/` — CORS headers, static file serving with MIME sniffing
- `lib/` — shared utilities (ID generation, ordering, sitemap I/O)

### Client (`client/`)
- `shell.html` + `shell.js` — single-project dev workspace (tree nav + iframe preview + doc panel + prompt box)
- `home.html` + `home.js` — project management homepage (list/create/import projects)
- `preview-index.html` + `preview-app.js` — lightweight preview mode (static rendering, no dev tools)
- JS modules: `tree-nav.js` (directory tree), `doc-panel.js` (Markdown docs viewer/editor), `prompt-box.js` (prompt assembly & copy), `export-modal.js` (export/publish), `inspector.js` (iframe element inspection), `api-client.js` (frontend API wrapper)
- CSS: `shell.css` (theme + layout), `home.css` (homepage styles)
- `assets/marked.min.js` — vendored Markdown renderer

### CLI (`bin/`)
- `axhost-make.js` — command dispatcher
- `axhost-init.js` — workspace initialization (creates `projects/`, `.projects.json`, start scripts, `package.json`)
- `axhost-update.js` — syncs templates + static assets from `templates/` into workspace projects
- `axhost-build.js` — regenerates `templates/preview/` from `client/` sources
- `axhost-add-*.js` — scaffolding commands for pages, components, folders, docs
- `axhost-migrate.js` — data migration utilities
- `lib/helpers.js` — shared CLI helpers

### Templates (`templates/`)
- `preview/` — built output from `client/` (generated by `build`, consumed by `update`)
- `project/` — scaffolding templates for new prototype projects (agents.md, rules/, page/component HTML templates)
- `start-script/` — `start.cmd`, `start.ps1`, `start.sh` templates synced to workspace on `init`/`update`

## Key conventions

### Modification workflow
1. Edit `client/` or `templates/` source files
2. If touching `preview-index.html`, `shell.css`, `icons.js`, or `preview-app.js`: run `build` first
3. Run `update --all` to sync changes to workspace projects
4. Refresh browser to verify

### Icon system
Use `<iconpark-icon icon-id="xxx" size="14" color="currentColor"></iconpark-icon>`. The loader is at `client/js/icons.js` (dev mode) or `resources/js/icons.js` (standalone). When updating the CDN URL, modify `client/js/icons.js` and run `update --all`.

### Scope boundaries
- Only modify files within `axhost-make/` (bin/, client/, server/, templates/, skills/, system-rules/)
- Never modify files outside `axhost-make/` without explicit user permission
- Never commit user project files (`prototype/`, `projects/`) into this repo

### No frameworks
No Vue/React/Angular in client or templates. Everything is vanilla HTML/CSS/JS to minimize LLM generation and maintenance cost.

### Git
This is a standalone Git repo. Commits should target `axhost-make/` directory. Never push without user approval. `.gitattributes` enforces LF line endings.

## Agent guidance for generated projects

The `system-rules/` directory contains specifications that guide AI agents when developing prototype pages and components within **user projects** (not this framework):

- `dev-spec.md` — icon loading, page navigation via postMessage, resource paths
- `components-spec.md` — component design principles, Web Components pattern, CSS/JS separation, page reuse conventions
- `mobile-frame-spec.md` — mobile/mini-program shell templates, class naming, overlay positioning

The `skills/axhost-make-cli.md` documents the CLI skill commands available to agents working inside user projects.

For detailed framework development guidelines (including AxHost integration, publish flow, and workflow details), see `agents.md`.
