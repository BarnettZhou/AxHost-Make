# 10. Docs Panel Markdown 扩展

docs-panel 在标准 Markdown 基础上做了以下扩展（"魔改"），全部实现在 `client/js/md-renderer.js`。

## 1. 彩色文字 `{#color text}`

**语法**：`{#颜色值 文字内容}`

支持的颜色格式：
- 颜色名：`{#red 红色文字}`、`{#blue 蓝色文字}`
- Hex 值：`{#dc3737 自定义颜色}`、`{#f0f 短 hex}`

```markdown
这是 {#red 红色文字}，这是 {#1677ff 蓝色文字}
```

实现方式：注册为 marked.js 的 inline extension（`colorText`），渲染为 `<span style="color: ...">`。

## 2. 警告块（Alert Blockquote）

**语法**：在引用块首行加 `[type]` 前缀，后跟空格和正文。

```markdown
> [info] 这是一条信息提示
> [success] 操作成功
> [warning] 请注意
> [error] 发生错误
> [default] 默认样式
```

五种类型对应不同颜色：

| 类型 | 左边框颜色 | 背景色 |
|------|-----------|--------|
| `[info]` | `#1677ff` 蓝 | `rgba(22,119,255,0.08)` |
| `[success]` | `#52c41a` 绿 | `rgba(82,196,26,0.08)` |
| `[warning]` | `#faad14` 黄 | `rgba(250,173,20,0.08)` |
| `[error]` | `#ff4d4f` 红 | `rgba(255,77,79,0.08)` |
| `[default]` | `#595959` 灰 | `rgba(89,89,89,0.08)` |

实现方式：在 marked.js 的 `walkTokens` 中解析 `[type]` 前缀，存入 `token._alertType`；在 `renderer.blockquote` 中根据类型添加 `axhost-alert axhost-alert-{type}` CSS class。CSS 在 `shell.css` 中定义。

## 3. 文档内部链接

**语法**：三种格式的文档链接，点击后在 docs-panel 内导航，不刷新页面。

```markdown
[同页面文档](readme.md)                   → 切换到当前页面的 readme.md
[跨页面文档](@page_hash/doc.md)            → 跳到指定页面的 doc.md
[跨组件文档](#component_hash/doc.md)       → 跳到指定组件的 doc.md
```

解析规则（`parseDocLink`）：
- `xxx.md`（不以 `@`/`#` 开头，不含 `/`）→ `same-page`，在当前 docs 列表中切换
- `@hash/doc.md` → `cross-page`，`type=page`，导航到目标页面后打开对应 doc
- `#hash/doc.md` → `cross-page`，`type=component`，同上

渲染时生成 `<a href="#" data-doc-link="..." data-doc-name="..." data-doc-type="..." data-doc-path="...">`。点击事件由 `doc-panel.js` 中的 `attachDocLinkHandler` 统一处理。

## 4. 图片路径快捷方式 `$filename`

**语法**：使用 `$` 前缀引用项目图片，无需写完整路径。

```markdown
![alt]($my-image.png)
```

渲染时 `$my-image.png` 自动解析为项目 `images/` 目录下的完整路径（`resolveImagePath`）。同样支持编辑器中粘贴图片时自动上传并插入该语法（`bindImagePaste`）。

## 5. 链接样式

所有 docs-panel 内的链接统一样式（`shell.css`）：

- 颜色：`#1677ff`
- 无下划线
- hover 时颜色减淡为 `#4096ff`
- visited 状态颜色不变（不回退为浏览器默认紫色）

覆盖范围：`.doc-content a`、`.doc-view a`、`.doc-preview a`、`.rule-viewer-content a`。

## 6. 编辑模式下的文档链接自动补全

在编辑模式下，输入 `](@` 或 `](#` 触发自动补全：

- **Stage 1**：输入 `](@` → 列出所有页面；输入 `](#` → 列出所有组件
- **Stage 2**：选择页面/组件后输入 `/` → 列出目标页面/组件下的所有 `.md` 文档

实现于 `doc-panel.js` 的 `bindDocLinkAutocomplete`，复用 `acCore` 通用自动补全组件。

## 7. 图片粘贴上传

编辑模式下，直接在 textarea 中粘贴剪贴板图片，自动上传到项目 `images/` 目录，并插入 `![image]($filename)` 语法。实现于 `doc-panel.js` 的 `bindImagePaste`。

## 关键文件

| 文件 | 职责 |
|------|------|
| `client/js/md-renderer.js` | marked.js 扩展注册 + 渲染逻辑 |
| `client/js/doc-panel.js` | 文档面板交互、链接导航、自动补全、图片粘贴 |
| `client/css/shell.css` | alert blockquote 样式、表格样式、链接样式 |
