# 文档格式规范

> 本项目 `docs/` 目录下的 Markdown 文件（如 `readme.md`、`notes.md` 等）支持的扩展语法。

Axhost-Make 使用 marked.js v15 渲染文档，在标准 CommonMark/GFM 语法基础上扩展了以下特性。

---

## 1. 彩色文字

在行内使用 `{#color 文字内容}` 为文字着色。

```markdown
{#red 红色文字}
{#dc3737 自定义 HEX 颜色}
{#blue 支持 **嵌套 Markdown** 语法}
```

**规则**
- `color` 可以是 CSS 颜色名（如 `red`、`light-blue`）或 HEX 值。
- HEX 值可带 `#` 也可不带；不带时自动补 `#`（如 `dc3737` → `#dc3737`）。
- 内部文字支持所有行内 Markdown（粗体、斜体、链接等）。
- 无效颜色值会原样输出，不做转换。

---

## 2. Alert 引用块

在引用块顶部添加标签，可将引用块渲染为主题色块。

```markdown
> [success]
> 操作成功提示

> [warning]
> 需要注意的事项

> [error]
> 错误说明
```

**支持标签**

| 标签 | 竖条颜色 | 背景 | 用途 |
|------|---------|------|------|
| `[default]` | `#595959` | `rgba(89,89,89,0.08)` | 默认/中性提示（不写标签时等同此样式） |
| `[info]` | `#1677ff` | `rgba(22,119,255,0.08)` | 信息提示 |
| `[success]` | `#52c41a` | `rgba(82,196,26,0.08)` | 成功提示 |
| `[warning]` | `#faad14` | `rgba(250,173,20,0.08)` | 警告提示 |
| `[error]` | `#ff4d4f` | `rgba(255,77,79,0.08)` | 错误提示 |

**规则**
- 标签必须位于引用块第一段开头，紧跟一个空格或换行。
- 无标签的引用块使用 `default` 样式（灰色竖条）。
- 背景色采用竖条颜色的低透明度版本，明暗主题下均可自然融合。

---

## 3. 文档间链接

在文档内链接到其他文档，支持三种写法：

```markdown
[需求说明](requirement.md)            ← 同一页面/组件内的其他文档
[首页文档](@a1b2c3d4/readme.md)       ← 跨页面跳转
[组件文档](#e5f6g7h8/readme.md)       ← 跨组件跳转
```

**规则**
- 同一页面内：直接写文档文件名，如 `readme.md`、`api.md`。
- 跨页面：用 `@` + 页面 hash + `/` + 文档名。
- 跨组件：用 `#` + 组件 hash + `/` + 文档名。
- 被引用的文档必须实际存在于对应目录的 `docs/` 文件夹中。

---

## 扩展性说明

以上语法由框架内置的 marked.js 扩展实现，**仅**在 Axhost-Make 的文档渲染环境中生效（开发模式 Shell 的文档面板、Preview 模式的文档面板）。

若需新增文档级扩展语法，应在框架层面统一维护（`axhost-make/client/js/md-renderer.js` 与 `preview-app.js`），项目代码中不可自行引入其他 Markdown 解析器或语法扩展。
